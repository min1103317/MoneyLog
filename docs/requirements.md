# 머니로그(MoneyLog) 요구사항 정의서

- 작성자: (이상민)
- 작성일: 2026-07-23
- 프로젝트 기간: 5일

## 1. 프로젝트 개요
- 한 줄 소개: 개인이 수입/지출을 기록하고 카테고리별·월별 통계를 보는 가계부 웹 서비스
- 대상 사용자: 로그인한 개인 사용자(내 데이터는 나만 접근)
- 목표: (예) 부트캠프 전 과정 복습 + EC2 실배포까지 완주

## 2. 유저 스토리
| # | 유저 스토리 (~로서 나는 ~하고 싶다, 왜냐하면 ~)                                               |
|---|------------------------------------------------------------------------------|
| US-1 | 사용자로서 나는 안전하게 회원가입 및 로그인하고 싶다. 왜냐하면 정보는 개인적이기 때문이다.                          |
| US-2 | 사용자로서 나는 수입 및 지출 내역을 추가,수정,삭제 하고 싶다. 왜냐하면 돈이 어디서 들어오고 어디로 나가는지 알아야 하기 때문이다.  |
| US-3 | 사용자로서 나는 월별, 유형별, 카테고리별로 내역을 조회하고 싶다. 왜냐하면 특정 조건에 맞는 거래내역만 빠르게 조회하고 싶기 때문이다. |
| ... |                                                                              |

## 3. 기능 요구사항 (MoSCoW)
| ID | 기능 | 설명 | 우선순위(Must/Should/Could) |
|----|------|------|:-----------------------:|
| F-01 | 회원가입/로그인 | JWT 발급, BCrypt |           Must          |
| F-02 | 카테고리 관리 | 시드 + CRUD |          Must           |
| F-03 | 거래내역 CRUD | 수입/지출 등록·조회·수정·삭제 |          Must           |
| F-04 | 목록 필터·페이징 | yearMonth/type/categoryId + page/size |          Must           |
| F-05 | 월별 통계 | 총수입·지출·잔액, 카테고리별 지출 |          Must           |
| F-06 | 인가 | 본인 데이터만 |          Must           |
| F-07 | 검증·예외 | 금액>0, 날짜 필수, {success:false, code, message, data:null} |          Must           |
| F-08 | 프론트 4종 | 로그인/목록/등록/통계 |          Must           |
| F-09 | Swagger | API 문서화 |         Should          |
| F-10 | 배포 | Docker+GitHub Actions+EC2 |          Must         |
| ... | (도전) |  |          Could          |

## 5. 범위(Scope)
| 구분 | 내용                                                        |
|------|-----------------------------------------------------------|
| 기본 과제 (In Scope) | jwt 자체 로그인, 가계부 기본 기능(CRUD), 월별 통계, EC2 인스턴스 베포           |
| 도전 과제 (Stretch) | 소셜 로그인 연동, 거래내역 엑셀 다운로드, github Actions 기반 CI/CD 파이프라인 구축 |
| 이번엔 안 함 (Out of Scope) | 오픈뱅킹 연동, 다국어 지원, 등등                                       |
